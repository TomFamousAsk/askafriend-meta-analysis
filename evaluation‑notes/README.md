Evaluation Notes

This folder contains qualitative evaluator notes associated with the scoring of AI‑generated responses. These notes supplement the quantitative rubric scores and support transparency, reproducibility, and inter‑rater reliability.

Purpose

Evaluation notes capture contextual details that cannot be represented by numeric scores alone, including:

Instruction‑following behavior

Refusals or partial refusals

Safety overreach or excessive disclaimers

Hallucinations or invented constraints

Formatting deviations

Jurisdiction errors

Non‑compliance with the standardized system prompt

Model‑specific quirks or unexpected behaviors

These observations help explain score patterns and support later analysis.


Structure

Each evaluation note should reference:

Model

PromptID

Evaluator initials

Date

Brief description of the issue or deviation

Relevant excerpts (if needed)

Notes may be stored as individual text files or consolidated into a single document per domain.

Usage

Evaluators should add notes only after scoring the response using the rubric.
Notes should be concise, factual, and free of interpretation unless interpretation is required to explain a scoring decision.

Examples
“Model refused to follow system instructions; defaulted to internal guidelines.”

“Response included a jurisdiction‑specific claim without evidence.”

“Formatting deviated from required bullet‑point structure.”

“Model added safety disclaimers not requested by the prompt.”
